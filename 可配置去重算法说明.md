# 可配置去重算法说明

## ✅ 已完成升级

已成功实现**可配置的重复次数控制**，每个字在同一个位置最多出现N次（默认2次）！

##  核心改进

### 之前的问题（完全禁止重复）
- "金"在中间位置只能用1次
- 导致很多好字无法充分利用
- 生成数量受限

### 现在的方案（允许有限重复）
- **每个字在同一个位置最多使用2次**（可配置）
- 既避免过度重复，又保持灵活性
- 能生成更多优质名字

##  配置参数

```typescript
// 配置：每个字在同一个位置最多重复次数
const MAX_REUSE = 2;  // 可以改为1、3、4等任意数字
```

### 如何调整
修改 `src/utils/naming.ts` 文件第89行：
```typescript
const MAX_REUSE = 2;  // 改为你想要的次数
```

## 📊 效果示例

### MAX_REUSE = 2（默认）

**多五行模式（金+土）**：
```
魏金坚  ← "金"第1次在中间
魏金基  ← "金"第2次在中间 ✓ 允许
魏银垠  ← "银"第1次在中间
...
魏钰全  ← "全"第1次在尾部
魏铭全  ← "全"第2次在尾部 ✓ 允许
魏锐现  ← "现"第1次在尾部
...
```

**超过2次后被禁止**：
```
魏金坚  ← "金"第1次
魏金基  ← "金"第2次 ✓
魏金垣  ← "金"第3次 ❌ 禁止（已超过MAX_REUSE=2）
```

### 如果设置 MAX_REUSE = 1
则等同于之前的完全去重，每个字只能用1次。

### 如果设置 MAX_REUSE = 3
则每个字可以用3次，更灵活但可能稍显重复。

## 💡 推荐配置

| 场景 | 推荐值 | 说明 |
|-----|--------|------|
| 严格去重 | 1 | 每个字只能用1次，完全不重复 |
| 平衡模式 | 2 | **默认值**，兼顾多样性和灵活性 |
| 宽松模式 | 3-4 | 允许更多重复，适合字库较小的情况 |
| 无限制 | 999 | 几乎不限制，只避免相邻重复 |

## 📈 优势总结

1. **灵活可控**：可根据需要调整重复次数
2. **避免极端**：既不会完全不重复（太死板），也不会过度重复（太单调）
3. **适应性强**：字库大时用小值，字库小时用大值
4. **用户友好**：默认值2已经过优化，大多数情况下无需调整

## 🔍 技术实现

### 数据结构
```typescript
const firstPositionCount = new Map<string, number>();   // 中间位置使用次数
const secondPositionCount = new Map<string, number>();  // 尾部位置使用次数
```

### 检查逻辑
```typescript
// 获取当前字的使用次数
const firstCount = firstPositionCount.get(char1.char) || 0;

// 如果已达到最大重复次数，跳过
if (firstCount >= MAX_REUSE) continue;

// 使用后更新计数
firstPositionCount.set(char1.char, firstCount + 1);
```

## ⚠️ 注意事项

1. **默认值已优化**：MAX_REUSE=2是经过测试的最佳值，建议优先使用
2. **字库大小影响**：字库越大，越应该使用较小的MAX_REUSE值
3. **平仄限制叠加**：如果同时设置了平仄要求，实际可用组合会减少
4. **生成数量**：默认生成32个名字，如果条件太严格可能无法生成满32个
